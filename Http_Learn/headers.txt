Headers:
Request Headers - cодержат информацию о запросе
    Host - Указывает имя хоста и порт, к которому нужно подключиться
    User-Agent - Информация о клиенте, который отправляет запрос (информация о браузере)
    Accept - Указывает серверу, какие типы данных клиент хочет получить в ответ
    Accept-Language - Указывает серверу, на каком языке клиент предпочитает получить ответ
    Authorization - Содержит информацию для аутентификации пользователя
    Cookie - Передает данные сессии или другой информации, сохраненной на клиенте, обратно на сервер

Response Headers - cодержат информацию ответа
    Content-Type - тип передаваемых/получаемых данных
    Content-Length - размер тела запроса/ответа в байтах
    Set-Cookie: Используется сервером для установки куки на клиенте
    Cache-Control: Указывает правила кэширования содержимого
    Location: Используется для перенаправления клиента на другой URL
    WWW-Authenticate: Сообщает клиенту, что для доступа к ресурсу требуется аутентификация.

Entity Headers - информация об содержимом (длина или тип)
    Content-Encoding - определение было ли применено какое-то средство компрессии (например gzip)
    Content-Disposition - указывает клиенту как необходимо отображать файл (inline / attachment / form-data)
    Content-Language - язык содержимого

General Headers - общая инфа
    Connection: Управляет поведением соединения.
    Upgrade: Используется для обновления протокола на другой
    Via: Информация о прокси-серверах, через которые прошел запрос или ответ.


MIME-типы (типы контента Content-Type)
    text/ — тип для текстовых данных (html,css etc.)
    application/ — для данных, которые обрабатываются программами (json, xlml, pdf etc.)
    image/ — для передачи изображений
    audio/ — для аудиофайлов
    video/ — для видеофайлов
    multipart/ — для отправки нескольких частей данных
    font/ — для шрифтов
    message/ — для сообщений
    model/ — для 3D моделей
